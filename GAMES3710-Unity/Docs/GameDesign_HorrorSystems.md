# 恐怖游戏系统设计文档

## 技术栈
- Unity 6.3 URP
- Starter Assets - First Person Controller

---

## 核心系统概览

### 1. 敌人AI系统
- **数量**: 同时仅1个敌人
- **移动**: NavMesh + Waypoint混合
- **状态机**:
  - **巡逻**: Waypoint路径片段池，终点随机选下一片段
  - **追逐**: NavMesh实时寻路，视线检测到玩家触发
  - **搜索**: 丢失玩家后，以最后可见位置为中心NavMesh随机采样游荡，超时回归巡逻
- **搜索→追逐**: 直接切换，无渐进过程

### 2. 感知系统
- **视线检测**: FOV角度判断 + 单射线遮挡检测
- **渐进发现**: 无，直接触发追逐
- **躲藏点**: 玩家在躲藏点内不参与检测
- **扩展预留**: 声音吸引（暂不实现）

### 3. 躲藏点系统
- **进入方式**: 统一按键交互进入
- **安全性**: 绝对安全，敌人不检查
- **视角**: 初期完全锁定，后续可扩展为有限转动
- **退出**: 玩家可随时按键退出

### 4. 资源管理系统
- **San值**:
  - 降低方式: 时间自然流逝
  - 效果: 低San值触发后处理滤镜
  - 归零: 游戏结束
  - 恢复: 消耗品道具
- **电池/夜视**:
  - 开关: 按键切换，开启时持续消耗
  - 耗尽: 直接关闭
  - 充电: 拾取电池自动充满

### 5. 锁钥/解谜系统
- **结构**: 线性流程
- **物品栏**: 无UI，内部状态记录
- **交互**: 靠近按键，自动检测是否持有所需道具
- **类型**: 钥匙开门、机关操作（水泵等）

## 游戏结构

### 死亡与存档
- **被抓**: 游戏结束
- **存档**: 按关卡自动存档

### 关卡设计
| 关卡 | 类型 | 核心玩法 |
|---|---|---|
| 1 | 探索 | 无敌人，纯收集解谜 |
| 2 | 潜行 | 室内巡游追逐 + 收集解锁 |
| 3 | 逃跑 | 高速追逐，选择路线避障，到达固定出口 |

### 关卡3详细
- **视角**: 第一人称
- **移动**: 自动向前跑
- **场景**: 预设路线与障碍物
- **操作**: 按键选择左/右转向或偏移
- **失败条件**:
  - 撞障碍物: 减速，累计2次被追上→游戏结束
  - 进入绝路: 直接游戏结束
- **设计意图**: 记忆正确路线 或 运气
- **资源系统**: 无San值/电池

### 系统与关卡对应
| 系统 | 关卡1 | 关卡2 | 关卡3 |
|---|:---:|:---:|:---:|
| 敌人AI | ✗ | ✓ | 特殊(追逐者) |
| San值 | ✓ | ✓ | ✗ |
| 电池/夜视 | ✓ | ✓ | ✗ |
| 锁钥解谜 | ✓ | ✓ | ✗ |
| 躲藏点 | ✗ | ✓ | ✗ |

### 场景光照
- 场景有明亮/黑暗区域分布
- 玩家需策略性使用夜视相机

## 实现方案决策

### 敌人AI - 巡逻系统

**Waypoint结构**: 方案B（路径为父物体，子物体为路径点）

**实现步骤**:
1. `PatrolPath` - 路径片段组件，自动读取子物体作为路径点，配置下一个可选路径列表
2. `EnemyAI` - 敌人控制器，状态机骨架（Patrol/Chase/Search），NavMeshAgent驱动
3. 巡逻逻辑 - 沿路径移动，到达终点随机选下一片段
4. 场景测试验证
